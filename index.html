Project Idea

Whilst playing the 2018 game ‘A way out’ with a friend of mine there was a mini game of connect 4 that we played. I proceeded to lose this game more than 10 times in a row, to much frustration. On several occasions since then I have still failed to win, so I conclude that I am terrible at this game. Because I believe everyone should feel the sting of defeat, I am sure that it would be entertaining to create unwinnable games that everyone would lose.

I am proposing to create one or a collection of unwinnable games. They would keep track of successive defeats, or perhaps time without a win. It could also laugh or mock the player in some way after each separate loss.

In order to create unwinnable games (without cheating), we must use games that have been ‘solved’. Meaning there is a perfect way to play that ensures a certain outcome, usually a win or draw. Connect 4 is one such game, others include Tic-tac-toe and checkers, which have varying degrees of complexity. If creating a collection of unwinnable games, a menu would be needed to select and individual game to play, this would also be a sensible place to display the statistics such as losing streaks.

Using connect 4 as an example it would appear like the existing game, in a digital space. This involves a 7x6 grid in which players take turns in placing tokens in any available column that would ‘fall’ to the lowest available row. The winner being the 1st to create an unbroken line of 4 of their tokens vertically, horizontally, or diagonally. The game could also function at this point as a multiplayer game where people play against each other if that functionality was desired. The solution to the game in this case it means that by perfect play the player who goes 1st will always win. Our bot will therefore go first and follow the necessary steps to produce perfect play. These steps would be dictated by a mathematical proof that must be converted into a structure of code.

I believe that software such as unity may be a way to create such a thing. Unity is free to use for students and therefore accessible for the purposes here. As solving the game individually would prove both difficult and time consuming, one would presumably refer to an existing solution such as Victor Allis’s “A Knowledge-based Approach of Connect-Four”.

The skills required for such a task would include some proficiency with unity which has many online tutorials. Additionally, some mathematically literacy would also be assumed to understand the solution to some of the more complex games. Finally experience programming in whatever the applicable language is would be needed to translate the strategy into a decision-making algorithm. Learning resources for mainstream languages are readily available online.

The simplest version of this product would be a Tic-Tac-Toe bot that plays perfectly to never lose. There is a large scope for growth with a menu system and several other mini games that could be added to create a collection of more complex games.
